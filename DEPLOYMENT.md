## 📝 How to Update Your Site

### Making Changes

1. Edit any HTML, CSS, or asset files locally
2. Test locally using VS Code Live Server or:
   ```bash
   python -m http.server 8000
   ```
3. Commit and push changes:
   ```bash
   git add .
   git commit -m "Your update description"
   git push
   ```
4. Both GitHub Pages and Vercel will **auto-deploy** within 1-2 minutes

### Replacing Placeholder Images

The `assets/` folder currently has SVG placeholders. To use real images:

1. Replace files in `assets/` folder:
   - `book-1.jpg`, `book-2.jpg`, `book-3.jpg` (book covers)
   - `hero.jpg` (hero/banner image)
2. Keep the same filenames for automatic compatibility
3. Recommended sizes:
   - Book covers: 400x600px
   - Hero image: 1200x600px or larger

#### For GitHub Pages:
1. Buy a domain (e.g., islamicbookstore.com)
2. Go to repo Settings → Pages → Custom domain
3. Add your domain and configure DNS:
   ```
   Type: CNAME
   Name: www
   Value: muhammadsamiullah0786.github.io
   ```

#### For Vercel:
1. Run `vercel domains add yourdomain.com` or use dashboard
2. Configure DNS with provided records
3. SSL auto-configured

## 🚨 Troubleshooting

### GitHub Pages not loading?
- Wait 2-3 minutes after first deployment
- Check build status: https://github.com/muhammadsamiullah0786/islamic-book-store/deployments
- Verify GitHub Pages is enabled in Settings → Pages

### Vercel deployment issues?
- Check deployment status: https://vercel.com/muhammad-sami-ullahs-projects-67ef39ce/website
- View build logs in Vercel dashboard
- Ensure Git is pushing successfully

### CSS or images not loading?
- All paths should be relative (no leading `/`)
- Check browser console for 404 errors
- Verify file names match exactly (case-sensitive on Linux servers)
